《深入理解Java虚拟机》 笔记（一）

发表于 2020-10-26 更新于 2020-10-30 分类于 个人计划

一、自动内存管理

1.Java内存区域与内存溢出异常（1d）

1）运行时数据区域

程序计数器

  • 用于控制程序的走向，是线程私有的。
  • JVM的字节码解释器通过改变程序计数器的值来获取下一行执行的代码，程序的分支，循环，跳转，异常处理以及线程恢复都需要依赖计数器。
  • 当程序计数器指向一个Java方法时，指向的是Java方法对应的直接地址；当指向一个本地方法时，计数器的值为null，这是Java虚拟机规范中唯一一个没有OOM的区域。

Java虚拟机栈

  • 用于控制方法执行的顺序，生命周期与线程相同，是线程私有的。
  • 线程中每当有一个方法被执行时，都会有一个栈帧被创建并被压入虚拟机栈中，每个栈帧都会存储方法的局部变量表，操作数栈，动态常量以及方法出口
  • 如果栈帧的深度大于虚拟机规定的最大值，则会报StackOutExcepiton；栈帧的深度可以扩展，如果扩展不了则会报OOM。

Java本地方法栈

  • 跟虚拟机栈基本相同，只是服务对象不同，本地方法栈服务于本地方法
  • HotSpot中将两者放到了一起

Java堆

  • JVM用于存储对象实例的地方，是线程公有的。
  • Java内存管理最主要的地方，也称为GC堆，Jdk8之前基本采用分代算法，分为新生代，老年代
  • 内部存在TLAB（本地方法缓存），这是堆为每个线程分配的一块区域，用于线程创建对象使用，提高了内存分配的效率，方便管理。

方法区

  • 用于存储类型信息，常量以及静态变量，是线程公有的。
  • HotSpot在jdk8之前使用永久代实现，jdk8后使用元空间替代

运行时常量池

  • 用于存储运行时产生的常量

直接内存

  • jdk1.4引入了NIO，这是一种基于通道的IO方式，可以直接在JVM管理范围外进行直接的内存分配。

使用git将本地文件上传到github仓库

发表于 2019-07-10 更新于 2019-07-11 分类于 git

1.linux下安装git

2.生成公钥

`ssh-keygen -t rsa -C "yourEmail@email.com"`

将邮箱替换成自己的邮箱后，输入命令，会提示你以下信息

Enter file in which to save the key (/root/.ssh/id_rsa):
Enter passphrase (empty for no passphrase):
Enter same passphrase again:

不输入直接按enter键，则按照默认的方式生成公钥，默认生成地址为括号中提示的地址 ___/root/.ssh/id_rsa___

3. 添加公钥到GitHub

打开GitHub，登录自己的账号在 Setting 里找到 SSH and GPG keys 选项卡，点击 New SSh key 按钮。
填写标题，复制刚刚生成的公钥 _（文件名称为id_rsa.pub）_ 填写到key中，然后点击 Add SSH Key 按钮。

4. 测试是否能够连接

回到linux系统，输入命令：

ssh -T git@github.com

提示以下信息：

Are you sure you want to continue connecting (yes/no)?

输入yes，如果成功会显示：

Hi XXX ! You’ve successfully authenticated, but GitHub does not provide shell access.

5. 设置用户信息

输入以下命令设置用户名和Email

git config --global user.name " yourUserName"
git config --global user.email "yourEmail"

6. 创建远程仓库和本地仓库并将其关联

1）创建远程仓库

打开GitHub，点击 Your Repositories ，新建一个远程仓库，并复制远程仓库的地址。

2）创建本地仓库

回到linux，打开想要创建仓库的文件位置，输入以下命令：

git init

输入命令后，会在当前文件夹下创建一个 .git 文件夹，用于存储所有版本，标记信息

3）关联本地，远程仓库

在linux下，输入命令：

git remote add origin https://yourRepo

https://yourRepo 换成刚刚复制的远程仓库地址

7. 提交本地文件，并推到远程

输入以下命令：

给提交的文件或文件夹添加标记
git add 文件位置

提交本地文件到head
git commit -m "提交描述"

将已经提交的文件推到远程
git push origin master

hexo个人博客实践2

发表于 2019-07-09 更新于 2019-07-11 分类于 hexo实践

1.资源文件夹

资源（ASSet）指 source 文件夹中除文章外的所有文件。

例如： 将图片放到 source/images 文件夹中，在通过 ![](/images/image.jpg) 的方式访问

文章资源文件夹

将 _config.yml 文件中的 post_asset_folder 选项设置为 true，就可以打开文章资源文件夹。

此功能打开时，每次输入 hexo new title 命令，都会生成一个与markdown文件同名的文件夹，这个文件夹中的资源文件可以通过相对路径进行链接。
